

# 配送管理システム 技術仕様書 (SPEC.MD)

## 1. プロジェクト概要

配送現場の経験値とデジタル技術（Google Maps API, 天気・Windy.com API）を融合させ、効率的かつ柔軟な配送計画を策定するWebベースのデリバリーシステム。

## 2. デザイン・UIガイドライン

* **コンセプト**: プロフェッショナルなビジネスカラー、視認性重視、直感的な操作性。
* **カラーパレット**:
* 背景色: ホワイト (`#FFFFFF`)
* ブランド・アクセントカラー: 濃紺 (`#000080`)


* **形状定義**:
* 角丸禁止（すべてのコンポーネント、ボタン、枠は `border-radius: 0px` を厳守）



## 3. 機能要件詳細

### 3.1 マスター管理 (CRUD)

* **トラックマスター**: 社名、ドライバー名、車番、最大積載重量の管理。
* **向け地/出発地マスター**: 郵便番号、住所、電話番号の登録。
* **データ整合性**: すべてのマスターデータは、配車計画の基盤として厳密に管理する。

### 3.2 配車計画機能

* **計画入力**: 日付、出発地、到着地、宵積みフラグ、積み下ろしバッファ（15分/30分単位設定）の設定。
* **コスト・効率計算**: 高速道路利用の有無による時間・コストのシミュレーション。
* **ドライバーナレッジ学習**: 特定ドライバーの優先ルート設定機能（AIがルート最適化の際に、そのドライバーの傾向を優先的に加味する）。

### 3.3 手動調整機能

* **柔軟性**: AIによる自動最適化の結果に対し、管理者が配送順序をドラッグ＆ドロップまたはリスト操作で手動入れ替え可能にする。

### 3.4 ドライバー連携

* **リアルタイム同期**: ドライバー向けアプリへ、最新の配車予定とルートを即時同期。
* **位置情報活用**: アプリ起動時のみGPS位置情報を取得し、ダッシュボードへフィードバック。
* **運行報告**: 出発、積み込み完了、到着、荷降ろし完了のステータス報告機能。

## 4. 外部API・連携基盤

* **Google Maps Platform**: 地図表示、ルート最適化、ETA（到着予想時刻）取得。
* **気象情報API**: 配車当日の運行リスク管理（天気予報取得）。
* **Windy.com API**: 現場の気象状況およびライブカメラ連携による状況把握。

## 5. データ構造案 (リレーショナルモデル)

| テーブル名 | 主要属性 |
| --- | --- |
| `trucks` | ID, 社名, ドライバー名, 車番, 積載重量 |
| `locations` | ID, 種別(出発/向け地), 郵便番号, 住所, 電話番号 |
| `deliveries` | ID, トラックID, 出発地ID, 日付, 宵積みフラグ, 高速フラグ, バッファ時間 |
| `delivery_routes` | ID, 配車ID, 拠点ID, 順序(手動調整対象) |
