# 配送管理システム

配送計画、マスター、運行状況、ドライバー報告をブラウザ上で一元管理する React 製 Web アプリです。

- 公開版: https://tsuyo39-glitch.github.io/delivery-system/
- 仕様: [SPEC.MD](./SPEC.MD)

## 主な機能

- 運行ダッシュボード: 配車件数・運行状況・データ不整合の確認、表示中データの CSV 出力
- 配車計画: 配車の作成・複製・削除、配送順序の入れ替え、ドラッグ&ドロップ、配車指示書の印刷
- マスター管理: トラック、出発地、向け地の登録・編集・削除
- ドライバー連携: 運行ステータスと位置情報のモック報告、報告履歴の CSV 出力
- 外部 API モック: ルート、天候、現場情報の確認
- データ管理: バックアップ JSON の作成・復元、サンプルデータ復元、参照整合性の検証

## 基本的な使い方

1. 「トラックマスター」で車両とドライバーを登録します。
2. 「出発地/向け地マスター」で配送拠点を登録します。
3. 「配車計画」で日付、トラック、出発地、配送条件を入力して配車を作成します。
4. 作成した配車を選び、向け地の追加と配送順序の調整を行います。
5. 「運行報告」で出発、積み込み完了、到着、荷下ろし完了を報告します。
6. 「運行ダッシュボード」で全体の進捗とデータ検証結果を確認します。

同じ日付・同じトラックの配車は、警告内容を確認してから登録できます。配車が参照しているマスターは削除できません。

## データの保存とバックアップ

入力データは、利用中のブラウザの `localStorage` に保存されます。サーバーや他の端末とは自動同期されません。ブラウザのデータ削除、別ブラウザへの移行、端末変更の前には「データ管理」でバックアップ JSON を作成してください。

バックアップ JSON とサンプルデータの復元は、現在のブラウザ内データを置き換えます。画面の確認パネルで対象を確認してから実行してください。

## ローカル開発

必要環境は Node.js と npm です。

```bash
npm install
npm run dev
```

開発サーバーの URL はターミナルに表示されます。ブラウザは自動起動しません。

## 検証

変更後は次のコマンドを実行します。

```bash
npm run build:pages
npm test
rg -n "border-radius" src
```

`npm run build:pages` は TypeScript と Vite のビルド後、GitHub Pages 用の `docs/` とルートの `index.html` を同期します。デザイン規約により、`border-radius` は `0` のみ使用できます。
